# Proposition de fondation du projet

## Objectif

Proposer une base propre pour démarrer le projet forum en équipe.

## Structure proposée

- `cmd/` : point d'entrée du serveur Go
- `internal/` : logique interne de l'application
- `web/` : fichiers liés à l'interface web
- `docs/` : documentation du projet

## Points déjà préparés

- README de base
- Cahier des charges
- Structure initiale du projet
- Module Go initialisé
- Convention de commits

## Décisions à prendre en équipe

- Choix des catégories du forum
- Schéma de base de données
- Répartition des fonctionnalités
- Organisation Trello
- Design général du site
- Règles de branches et Pull Requests

## Point d'arrêt proposé

Cette branche peut être présentée au groupe comme base de départ.

On peut valider ensemble :
- la structure du projet ;
- le cahier des charges ;
- la convention de commits ;
- l'organisation Trello ;
- la répartition des tâches.

On ne doit pas encore coder seul :
- l'authentification ;
- les sessions ;
- le schéma final de base de données ;
- les pages HTML finales ;
- le design complet ;
- les routes de création, modification et suppression de posts.

## Proposition de priorités Trello

- Documentation et cahier des charges
- Architecture Go et routing
- Base de données SQLite
- Authentification et sessions
- Posts et commentaires
- Likes, dislikes et filtres
- Interface responsive et accessibilité
- Tests et correction des erreurs
